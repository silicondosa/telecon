\doxysubsection{Detailed Description}
This is the primary user-\/level entry point for all \mbox{\hyperlink{class_telecon}{Telecon}} behavior. This class creates and manages windows, which in turn create and manage lower-\/level items such as charts and plots.

There should only ever be one \mbox{\hyperlink{class_telecon}{Telecon}} object active (started) at a time. After a \mbox{\hyperlink{class_telecon}{Telecon}} object is stopped or deleted, it is safe to start another if necessary. However, each object can only be run once. After the object is stopped, it may not be started again.

\mbox{\hyperlink{class_telecon}{Telecon}} is inherently thread-\/safe with regard to the controller (calling) thread and the UI thread (which is created and managed by \mbox{\hyperlink{class_telecon}{Telecon}}). However, if a \mbox{\hyperlink{class_telecon}{Telecon}} object is used by multiple controller threads, external synchronization must be used to prevent concurrent access to member functions.
