Detailed Description

This is the primary user-level entry point for all Telecon behavior. This class creates and manages windows, which in turn create and manage lower-level items such as charts and plots.

There should only ever be one Telecon object active (started) at a time. After a Telecon object is stopped or deleted, it is safe to start another if necessary. However, each object can only be run once. After the object is stopped, it may not be started again.

Telecon is inherently thread-safe with regard to the controller (calling) thread and the UI thread (which is created and managed by Telecon). However, if a Telecon object is used by multiple controller threads, external synchronization must be used to prevent concurrent access to member functions.
